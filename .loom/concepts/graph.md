# Graph

문서 간 위키링크 관계를 표현하는 방향 그래프.

## 목적

LLM이 전체 문서를 읽지 않고도 문서 간 관계를 파악할 수 있도록, 위키링크를 파싱하여 구조화된 관계 정보를 제공한다.

### 핵심 시나리오

- 특정 문서의 outgoing links(이 문서가 참조하는 문서들)를 조회한다
- 특정 문서의 incoming links(이 문서를 참조하는 문서들, 역참조)를 조회한다
- N-hop 이웃을 조회하여 문서 주변의 관계망을 탐색한다
- 두 문서 사이의 연결 경로를 탐색한다
- 전체가 아닌 필요한 부분만 반환하여 토큰을 절약한다

## 동작 원리

### 역할

- 노드: 마크다운 문서. 파일 경로 또는 문서명으로 식별
- 엣지: 위키링크(`[[target]]`). 문서 A가 문서 B를 참조하면 A → B 방향 엣지

### 핵심 동작

- 구축: [[source]]가 제공한 문서들에서 위키링크를 파싱하여 노드와 엣지를 생성
- 관계 탐색: 특정 노드의 in/out 엣지, N-hop 이웃 반환
- 경로 탐색: 두 노드 사이의 연결 경로 탐색
- 부분 조회: 쿼리에 해당하는 서브그래프만 반환

## 트레이드오프

- 문서의 내용(본문 텍스트)은 다루지 않는다 — 관계 구조만 취급
- 위키링크 외의 참조 방식(URL 링크, import 등)은 파싱하지 않는다

## 관련 컨셉

- [[source]] - 그래프 구축의 입력 범위를 제공한다
- [[index-store]] - 구축된 그래프를 디스크에 영속화한다
- [[search]] - 검색 결과를 진입점으로 받아 그래프 탐색을 수행한다
