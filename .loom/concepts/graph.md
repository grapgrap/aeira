# Graph

문서 간 위키링크 관계를 표현하는 방향 그래프.

## 목적

LLM이 전체 문서를 읽지 않고도 문서 간 관계를 파악할 수 있도록, 위키링크를 파싱하여 구조화된 관계 정보를 제공한다.

### 핵심 시나리오

- sync 시 변경된 문서의 위키링크를 증분 파싱하여 그래프를 갱신한다
- 조회 시 SQLite에서 그래프를 복원하여 관계를 탐색한다

## 동작 원리

### 역할

- 노드: 마크다운 문서. ir의 documents 테이블 경로로 식별
- 엣지: 위키링크(`[[target]]`). 문서 A가 문서 B를 참조하면 A -> B 방향 엣지
- dangling 노드: 엣지의 대상이지만 실제 문서가 존재하지 않는 노드

### 핵심 동작

- 구축 (sync): [[index-store]]가 감지한 변경 문서에서 위키링크를 파싱하고, 이름 인덱스(파일명 stem -> 경로)로 대상을 해석하여 엣지를 증분 저장
- 복원 (load): [[index-store]]의 aeira_edges 테이블에서 전체 엣지를 읽어 인메모리 Graph로 복원
- 조회: 세 가지 프리미티브로 관계 정보를 반환
  - neighbors(node, direction?) -- 1-hop 이웃. 방향은 outgoing/incoming/both
  - path(from, to) -- 두 노드 간 모든 경로. 중간 노드를 포함하여 N-hop 관계를 드러냄. 노드 재방문을 금지하여 순환을 차단하고, 경로 수에 상한을 둔다
  - snapshot() -- 전체 노드 + 엣지 목록

## 트레이드오프

- 문서의 내용(본문 텍스트)은 다루지 않는다 -- 관계 구조만 취급
- 위키링크 외의 참조 방식(URL 링크, import 등)은 파싱하지 않는다

## 관련 컨셉

- [[vault]] - 그래프 구축의 입력 범위를 제공한다
- [[index-store]] - 그래프의 엣지를 영속화하고, 변경 감지와 복원을 담당한다
- [[search]] - 검색 결과를 진입점으로 받아 그래프 탐색을 수행한다
