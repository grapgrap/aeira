# Search

aeira의 문서 탐색 진입점. ir에 위임하여 문서를 찾고, 결과를 [[graph]] 탐색으로 연결한다.

## 목적

사용자와 소비자(스킬)가 항상 검색으로 시작할 수 있도록 통합된 진입점을 제공한다. 검색 결과에서 그래프 탐색으로의 전환을 aeira 내부에서 처리하여 소비자의 통합 부담을 제거한다.

### 핵심 시나리오

- 키워드를 검색하면 해당 키워드를 포함하는 문서 목록을 반환한다 (BM25 + 벡터 + 리랭킹)
- 문서 이름으로 검색하면 해당 문서를 찾아 반환한다
- 검색 결과에서 그래프 탐색으로 연결하면 관련 문서의 맥락을 확장할 수 있다

## 동작 원리

### 역할

- 진입점: 모든 문서 탐색의 시작 지점
- ir 위임: 내용 검색을 ir CLI(`ir search --json`)에 위임
- 연결: 검색 결과의 path를 ir SQLite의 documents 테이블 경로로 그래프 노드에 연결

### 핵심 동작

- 내용 검색: ir CLI를 호출하여 하이브리드 검색 수행 (BM25, 벡터, 시맨틱)
- 문서 탐색: ir SQLite의 documents 테이블에서 이름/경로 기반 조회
- 결과→그래프 연계: 같은 SQLite 내 edges 테이블과 JOIN하여 관련 문서 탐색

## 트레이드오프

- 검색 품질과 성능은 ir에 의존한다 — 자체 검색 인덱스를 구축하지 않는다
- ir가 설치되어 있어야 검색 기능이 동작한다
