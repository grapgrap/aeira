# aeira

위키링크 기반 문서 관계 그래프 도구. Obsidian의 탐색 강점을 추출하되, 앱 종속과 메타데이터 잔류를 제거한다.

## 컨셉 목록

- [[vault]] — 이름으로 식별되는 문서 관리 단위. 문서 수집 경계 + 생명주기 관리
- [[graph]] — 문서 간 위키링크 관계를 표현하는 방향 그래프
- [[index-store]] — 구축된 그래프의 디스크 영속화
- [[search]] — 문서 탐색의 진입점, 외부 도구 위임 + 그래프 연계

## 관계

```
search ──▶ vault ──▶ graph ──▶ index-store
(진입점)     (관리/수집)  (구축/탐색)   (영속화)
  │                        ▲         │
  └────────────────────────┘         │
      (검색 결과 → 그래프 순회)        │
                     vault ◀─────────┘
                     (변경 감지)
```

- search가 사용자의 질의를 받아 vault 범위 내에서 외부 도구로 문서를 찾는다. vault가 문서 범위를 한정하고, graph가 위키링크를 파싱하여 관계를 구축하며, index-store가 이를 프로젝트 외부에 캐싱한다. 검색 결과는 graph의 진입점으로 연결되어 관계 탐색으로 확장된다.
- index-store는 vault의 문서 변경을 감지하여 인덱스 유효성을 판단한다.
