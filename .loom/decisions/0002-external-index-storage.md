# 프로젝트 외부 인덱스 저장

## 맥락

Obsidian이 `.obsidian/` 폴더를 프로젝트에 남기는 것이 주요 불만이었다. aeira의 인덱스를 어디에 저장할지 결정이 필요했다.

## 결정

인덱스를 대상 프로젝트 외부에 저장한다. 프로젝트 내부에는 어떤 메타데이터도 남기지 않는다 (non-intrusive).

### 제약 조건

- Obsidian의 메타데이터 잔류가 기존 불만: 프로젝트에 도구 종속 파일을 남기면 안 됨
- git에 도구 메타데이터가 포함되면 안 됨: .gitignore 의존도 제거

### 전제 조건

- 프로젝트 경로로부터 인덱스 저장 위치를 결정론적으로 도출할 수 있다: 동일 source에 대해 항상 같은 캐시를 찾을 수 있어야 함
- 캐시 유실은 재구축으로 복구 가능하다: 캐시는 원본(문서)에서 파생된 데이터

## 대안

### 프로젝트 내부 저장 (.aeira/)

- 설명: 프로젝트 루트에 `.aeira/` 디렉토리를 만들고 .gitignore에 추가
- 트레이드오프: 접근이 쉽지만, non-intrusive 원칙 위반. .gitignore 관리를 사용자에게 위임해야 함

### 인메모리 전용 (저장 안 함)

- 설명: 매 호출마다 전체 파싱하고 디스크에 아무것도 저장하지 않음
- 트레이드오프: 가장 깨끗하지만, 문서가 많을 때 매 호출 비용이 커짐. CLI 우선 아키텍처와 조합하면 성능 병목
