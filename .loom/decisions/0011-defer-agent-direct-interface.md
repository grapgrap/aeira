# 에이전트 직접 접근 인터페이스 보류

## 맥락

aeira를 AI 에이전트가 더 쉽게 사용할 수 있도록 MCP 서버나 skill/plugin 형태의 인터페이스를 추가할지 검토했다. aeira는 AI 활용을 전제로 만든 도구이므로 AI 접근성을 높이는 것이 자연스러운 방향이었다.

## 결정

에이전트가 aeira에 직접 접근하는 인터페이스(MCP, plugin 등)를 지금 추가하지 않는다. 현재 에이전트는 loom/shulker 스킬을 통해 aeira를 사용하며, 스킬이 곧 에이전트-aeira 간 인터페이스 역할을 한다.

### 제약 조건

- 현재 aeira의 소비자는 loom/shulker 스킬뿐이며, 스킬이 Bash로 CLI를 호출하는 구조가 동작한다
- 에이전트가 스킬을 거치지 않고 aeira를 직접 사용하는 시나리오가 식별되지 않았다

### 전제 조건

- 스킬이 에이전트와 aeira 사이의 중개 역할을 충분히 수행한다
- 스킬 외부의 소비자가 등장하면 이 결정을 재검토한다

## 대안

### MCP 서버

- 설명: aeira를 MCP 서버로 감싸서 에이전트가 tool call로 직접 호출
- 트레이드오프: 타입이 있는 인터페이스와 자동 발견이 가능하지만, 스킬이 이미 중개하는 상황에서 불필요한 레이어가 추가됨. CLI 위에 얹으면 shell out 이중 비용, 코어 직접 import이면 라이브러리 분리 필요

### Skill/Plugin 배포

- 설명: aeira 사용법을 기술한 스킬을 별도로 만들어 배포
- 트레이드오프: aeira 코드 변경 없이 가능하지만, loom/shulker가 이미 그 역할을 하고 있어 중복
