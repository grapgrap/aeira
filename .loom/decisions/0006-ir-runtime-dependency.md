# ir 런타임 의존 방식

## 맥락

aeira가 ir을 CLI 호출 + SQLite 직접 접근으로 사용하기로 결정(0005)했으나, ir 바이너리를 사용자에게 어떻게 제공할지는 미결이었다. ir은 brew와 cargo로만 배포되며 라이선스가 명시되어 있지 않다.

## 결정

ir을 시스템 바이너리 런타임 의존으로 취급한다. aeira는 ir이 필요한 시점에 PATH에서 존재 여부를 확인하고, 없으면 설치 안내를 출력한다.

### 제약 조건

- ir의 라이선스가 명시되지 않아 재배포(번들링) 불가
- ir은 brew/cargo로만 배포: npm 패키지로 설치할 수 없음

### 전제 조건

- aeira 사용자가 brew 또는 cargo를 사용할 수 있는 환경: 개발자 대상 CLI 도구이므로 합리적 가정
- ir 없이도 그래프 구축(scan → parse → build)은 가능: ir이 필요한 것은 SQLite 저장과 검색 위임

## 대안

### postinstall 자동 다운로드

- 설명: ir이 GitHub Releases에 prebuilt 바이너리를 올려두면, npm postinstall에서 플랫폼별 바이너리를 다운로드
- 트레이드오프: 사용자 경험이 매끄럽지만, ir의 릴리즈 인프라에 의존. 라이선스 미명시 상태에서 재배포에 해당할 수 있음

### 바이너리 번들링

- 설명: ir 바이너리를 aeira 패키지에 포함하여 배포 (oxlint 방식)
- 트레이드오프: 설치가 간편하지만, 플랫폼별 바이너리 관리·라이선스·패키지 사이즈 문제. 배포 파이프라인 구축이 별도 프로젝트급
